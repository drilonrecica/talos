Lightweight, Coolify-aware monitoring for Docker servers.

Binnacle is a self-hosted, read-only monitoring dashboard for Linux servers running Docker workloads. Host metrics, logical service grouping, local SQLite history, and live updates — without a separate observability stack.

Why Binnacle?

Local-first

Core monitoring works offline. Data is stored locally in SQLite and no telemetry leaves your server by default.

Low overhead

A single Go binary, embedded frontend assets, and bounded collection keep resource use small on small servers.

Coolify-aware

Automatically groups Coolify and Compose services, while staying fully compatible with plain Docker containers.

Read-only by design

Binnacle never restarts, stops, deletes, or execs into containers. It only observes.

Live updates

Server-sent events push fresh host and resource snapshots to the web dashboard as they are collected.

Local history

Retained metric history and event logs are kept in SQLite with configurable retention and database budget limits.

Quick start

The supported path is Docker Compose. Set a setup token and the host Docker group ID, then start the service: 

export BINNACLE_SETUP_TOKEN=$(openssl rand -hex 32)
export DOCKER_GID=$(getent group docker | cut -d: -f3)
docker compose -f https://raw.githubusercontent.com/drilonrecica/binnacle/master/packaging/docker/docker-compose.yml up

Then open http://localhost:8080, complete setup, and sign in.
